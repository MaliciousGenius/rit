Programs to test

Following are descriptions of some simplified utilities. Read the specification carefully and then devise a set of tests to test conformance with the specifications. If the specification is incomplete, point this out and include tests to verify program behavior in these unspecified areas.

labcmp <file1> <file2>

Description

The labcmp utility compares two files. If the files are identical, labcmp will write a message to that effect on standard out. If they differ, it writes to standard output a message containing the number of lines that were the same. If one file is an initial subsequence of the other, that fact is noted. If exactly two file names are not specified on the command line a usage message is printed on standard output and the program terminates.

labgrep <searchString>

Description

The labgrep utility reads standard input and copies only lines that contains at least one occurance of the search string to standard output. If no search string is specified or there are too many arguments a usage message is printed on standard output and the program terminates.

labsort <column>

Description

The sort command sorts lines of standard input and writes the result on the standard output. The first character of a line is in column 0. The sort key is the decimal value of the longest sequence of digits in the line starting with the indicated column. If there is no digit in the indicated column then the sort key is zero.
 
For example, if the column number indicated in the command line argument is 5, here are some sample lines with their resulting keys:
LINE KEY
I am 72 years old today. 72
I am only 72 years old today. 0
I'm 72 years old today. 2
123456789 that's all folks! 6789

This is a stable sort, i.e., the order of lines with identical keys is not changed. If no column is specified or there are too many arguments a usage message is printed on standard output and the program terminates.

labsum (sum a list of numbers modulo 10n)

Description

The labsum utility calculates and prints the last n digits of the sum of the numbers read from standard input. A single argument is expected on the command line which is the number of digits to accumulate. If there is not exactly one command line argument specifying the number of digits to print a usage message is printed on standard output and the program terminates.

labtail (list the last n lines of a file)

Description

The labtail utility copies standard input to the standard output beginning at the nth to to last line. Copying begins at the nth to to last line and continues to the end of the input. Exactly n lines will be copied to standard output. If there is not exactly one command line argument specifying the number of lines to be output a usage message is printed on standard output and the program terminates.
